# Documentation Guidelines

## Documentation Style

- Use **single-line comments** for function documentation
- Avoid verbose multi-line documentation with `# Arguments`, `# Returns`, `# Examples` sections
- Keep documentation concise and focused on essential information only
- Prefer brief, clear descriptions over exhaustive explanations

## Rationale

- Verbose documentation creates maintenance overhead
- Code should be self-documenting through clear naming
- Brief comments are easier to read and maintain
- Reduces visual clutter in the codebase
